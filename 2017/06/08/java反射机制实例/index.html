java反射机制实例

发表于 2017-06-08

#java反射机制实例展示

例如：我们想要一个电脑运行。首要我们需要主板能够跑起来。

首先是主板类

public class MainBoard {
    public static void run(){
        System.out.println("MainBoard run...");
    }
}

应用程序运行类

public class ReflectTest {

    /**
     * 电脑运行
     * */
    public static void main(String[] args) {

        MainBoard mainBoard = new MainBoard();
        mainBoard.run();

    }
}

在这个基础上，如果我们想要添加声卡网卡或者其他的配件应用。原始的方法，我们需要去修改应用程序。而如果使用java反射机制的话，我们可以大大提高应用程序的可扩展性。

###应用java反射机制

1.我们可以定义一个接口，以备后面加入的新的配件应用的启动和关闭 

public interface PCI {

    public  void open();
    public  void close();

}

2.增加配置文件pci.properities，并将配置文件放至根目录下。

pci1=test.reflectDemo.类名      //类的路径名。

3.修改运行类，要读取配置文件

public class ReflectTest {

    /**
     * 电脑运行
     * */
    public static void main(String[] args) throws Exception {

        MainBoard mainBoard = new MainBoard();
        mainBoard.run();


        //扩展应用，读取配置文件
        File configFile = new File("pci.properities");

        Properties prop = new Properties();

        FileInputStream fis = new FileInputStream(configFile);

        prop.load(fis);      //将流数据加载到集合中

        for (int i = 0; i < prop.size(); i++) {

            String className = prop.getProperty("pci"+(i+1));   //读取pci1.

            Class clazz = Class.forName(className);            //获取配置文件中所声明的类。

            PCI p =(PCI) clazz.newInstance();

            p.open();

        }

    }
}

如果，要增加新的功能，就只需增加一个实现了PCI接口的类，并且在配置文件加入这个类的名字即可。

###应用程序的扩展

例如： 新增一个声卡，一个网卡。则只需实现PCI接口，并在配置文件中声明即可。

public class SoundCard implements PCI {

    public void open() {
        System.out.println("sound card run......");
    }

    public void close() {
        System.out.println("sound card close......");
    }
}

public class NetCard implements PCI {
    public void open() {
        System.out.println("net card run ....");
    }

    public void close() {
        System.out.println("net card close ....");
    }
}

修改配置文件：

pci1=test.reflectDemo.SoundCard
pci2=test.reflectDemo.NetCard

直接运行ReflectTest的main方法即可实现。
运行结果如下：

MainBoard run...
sound card run......
net card run ....
